01/ 08

本质抽象Essence

撕掉题面包装后，这道题是什么数学问题？

A 题 过滤设备监测

退化过程建模 + 剩余寿命预测 + 条件维护优化的复合问题，属于工业工程里的 CBM（Condition-Based Maintenance，基于状态的维护策略优化）。

三种数学成分交织：

  • 时间序列 / 退化过程——透水率作为带漂移、带季节性、带冲击的随机过程
  • 剩余寿命预测（RUL）——何时 per 跌破 37？
  • 随机动态规划 / 混合整数规划——维护时点与类型的决策

第 4 问是灵敏性分析 + 鲁棒优化。

B 题 台风降水过程

条件时空生成建模（conditional spatio-temporal generative modeling），属于数据驱动的物理代理模型——学习 P(降水时空场 | 台风路径、强度、环境) 这个条件分布并可从中采样。

三条可行路径：

  • 统计路径——EOF + 核/回归方法重构
  • 机器学习路径——U-Net / ConvLSTM / 树模型
  • 生成式路径——cGAN / Conditional Diffusion

陷阱提示：这不是纯预测题。Q2 要求对"有路径但无降水记录"的历史台风做条件采样；Q3 是分布外外推。

02/ 08

子问题拆解Structure

各子问题的链路与依赖关系

A 题 · 4 问 严格链式 Q1→Q2→Q3→Q4
  • Q1 数据清洗 + 三类维护效应识别 + 指标体系
  • Q2 固定维护规律下的寿命预测（需含预测区间）
  • Q3 最优维护方案使年均成本最小 （决策变量：每年大/中维护次数与时点）
  • Q4 成本扰动下方案稳健性 + 决策翻转阈值

关键耦合：Q1 指标体系会直接写死 Q2–Q4 的变量定义。常见失误：只给一个"下降速率"标量，正确做法是给出三元结构（趋势率 + 周期振幅 + 维护三元增益）。

B 题 · 3 问 链式 Q1→Q2→Q3，Q1 可并行
  • Q1a 台风特征工程（路径 / 强度 / 风速半径）
  • Q1b 降水分布指标（中心偏移 / 非对称性 / 带宽）
  • Q1c 特征↔降水的定量映射
  • Q1d 时空关系可视化（题目点名要）
  • Q2 为 KONG-REY、MAN-YI 生成降水场（无 GT）
  • Q3 虚拟台风情景 + 未来极端降水趋势

关键耦合：Q1 的特征体系必须倒推包含 Q3 情景所需变量——否则 Q3 要重搭模型。

04/ 08

核心难点Choke Points

按卡住概率降序列出——这三点决定成败

A 题 卡点集中在 Q3 的优化落地
60% · 中高风险 45% · 中风险 30% · 中低风险

#1 小维护的隐式建模（60%）
题目说每 3–5 天一次但没记录。你要决定是当"已知周期扰动"吸收进基线，还是用变点检测反推时点。前者简单后者严谨——队伍容易在这里纠结一天。

#2 Q3 的分式规划（45%）
年均成本 = (折旧 + 维护费) / 寿命，分子分母都依赖决策变量。处理：Dinkelbach 算法化线性、离散化 + DP、或直接启发式兜底。金融学类有天然优势——和分式投资组合是一个骨架。

#3 三类维护参数标定（30%）
每种维护的"提升幅度—衰减—季节依赖"至少 3 参数，三类共 9+。需要双指数衰减 + 非线性最小二乘。

B 题 三个卡点都可能致命
75% · 极高风险 65% · 高风险 55% · 中高风险

#1 GPM 数据的下载/读取/对齐（75%）
对没碰过气象遥感数据的队，光"读进来对齐到台风路径"可能卡 1.5–2 天。HDF5、半小时时间戳、经纬度投影、storm-centered 极坐标 转换——这一关不过全题停摆。

#2 Q2 生成模型选择（65%）
最大风险：选了深度生成模型但 72h 根本训不好。KNN 类比最保险但平庸；cGAN / Diffusion 创新高但需 GPU 且调参周期长。

#3 合理性论证（55%）
无 ground truth 如何证明生成对？需自设评估指标（与相似台风对比 / 物理约束 / 可视化）。设计不合理 = 直接压分。

06/ 08

常见失分点Pitfalls

历年阅卷高频扣分项

A 题 论文 > 建模 > 求解 > 格式
  • Q1 指标单薄——只给"月均下降速率"，应给趋势 + 周期 + 维护三元
  • Q2 无置信区间——寿命预测必须带预测区间
  • 三类维护合并处理——题目特意区分大/中/小
  • 年均成本公式写错——忘了 300 万折旧分摊到寿命年数。这是最致命的错
  • Q4 扰动幅度拍脑袋——"20% 浮动"无依据。用历史价格波动率给依据（金融学类顺手）
  • 缺关键图——至少 3 张：①完整曲线 + 维护标记；②维护前后对比；③最优方案 vs 当前方案成本柱图
B 题 地板分数极低的题
  • 数据处理不透明——没讲清楚 HDF5 怎么抽、怎么对齐
  • KONG-REY 与 MAN-YI 搞错——题目明确警告名称相似/时间重叠
  • 合理性指标缺失——Q2 只放几张图不给评估方法
  • 可视化敷衍——台风题"图为王"，无合格的路径叠加降水场图 = 直接压分
  • 物理约束违反——生成降水出现负值 / 尖刺 / 空间不连续
  • Q3 情景离谱——造出跨越大陆 2000 公里还加强的台风
  • 模型黑盒化——讲不清"为什么 MAN-YI 的降水应该长这样"

一句话总结Verdict

适合什么样的队伍？不适合什么样的队伍？

退化建模 ≈ 资产折旧
寿命预测 ≈ 违约预测
维护决策 ≈ 动态再平衡
Q4 ≈ 压力测试
— 金融学类的数学骨架 1:1 迁移

适合：有 1 名编程可靠成员（能写循环迭代 / 调 pulp）+ 熟统计的队伍。经济学直觉在 Q3、Q4 是加分项。

不适合：三人都没写过优化求解代码、或没耐心做数据清洗的队伍。

B 题 博天花板但风险极大
数据工程的前置成本
会吃掉你们大半时间
还没开始建模
论文周期就过半
— 金融学类选 B 的最大风险

适合：队内至少 1 人熟 Python 地学栈（xarray / cartopy / h5py）或有深度学习项目实战的队伍。最理想是大气 / 地理 / 遥感专业。

不适合：纯金融 / 经济 / 管理背景的队伍——强烈不推荐中财金融学类选 B，除非队里有 1 位 CS 或气象 / 地理背景的同学愿意全包数据处理。
